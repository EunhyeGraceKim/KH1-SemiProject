이주명 : include부분(header-로고+로그인 박스, nav-카테고리+장바구니링크, footer), 메인페이지, 상세페이지
김은혜 : 회원가입 페이지, 로그인 페이지, 아이디찾기 페이지, 비밀번호 찾기
정혜민 : 마이페이지(개인정보수정 페이지, 자신이 작성한 상품후기 목록페이지, 주문내역 페이지, 탈퇴페이지)
문상아 : 장바구니 페이지, 상품리스트 페이지, 관리자용 마이페이지
김진하 : 주문서 페이지, 결제팝업, 오류페이지


공통 페이지 크기부분 : 1700px(전체크기), 1080px(내용부분크기), 메인컬러
공통 기능 = 로그인 한 유저만 사용할 수 있도록 (제한:  주문하기 클릭시 로그인 판별 => 로그인 안되어있으면 로그인 페이지로)
======= 요구사항 ==========
1.header페이지(include)
내용: 로고 이미지, 회원가입+로그인 링크, navi(세일, 최신, 추천, 검색창, 장바구니 아이콘)

형태: navi컬러 : navi:li

기능: navi 전체 카테고리 부분 hover시 리스트down, navi hover시 fontcolor변화
장바구니 아이콘 - 상세페이지에서 장바구니 담기 클릭 시 메세지 출현 후 / 아이콘 변경(이 부분은 여유있을 때)



2.footer페이지(include)
내용: 고객문의 번호+문의 버튼=내용1 , 회사정보+링크=내용2, 인증내용=내용3
형태: 상단 div(내용1span / 내용2span) 하단 div(내용3)
기능: 회사정보 및 고객문의 클릭시 링크이동 




4.index페이지
내용: navi에 들어가 있는 항목(알뜰, 신상, 베스트) + MD추천 배너 이미지 4개씩 2번 총 8개 상품 슬라이드 형식으로 보여줌
형태: div로 각각 나눔
기능: 슬라이드 기능, 상품 클릭시 해당 상품의 상세내용 들어가기





5.회원가입 페이지
내용: 이름, id, 비밀번호, 비밀번호 확인, 이메일, 휴대폰, 배송주소, 성별, 생년월일, 약관동의, 중복+인증(이메일, id)

형태: table로 input[text] = 이름, id, 휴대폰 / input[password] = 비밀번호, 비밀번호 확인 / input[email] = 이메일
input[radio] = 성별 / input[text] = 년도 따로 월 따로 일 따로

기능: 유효성 검사, 중복확인, 주소검색(카카오api),  이용약관 팝업기능, 이메일 인증





6.로그인 페이지
내용: 아이디 비밀번호 입력, 아이디 비밀번호 찾기 링크, 로그인 버튼, 회원가입 버튼
형태: form태그
기능: 입력한 값을 DB에 전달 / 틀렸을 시 대화상자 출현 후 다시 로그인 페이지로

7.id찾기 페이지
내용: 이름, 이메일 입력
형태: form태그
기능: 입력한 값을 DB에 전달 

7-2. id찾기 결과 페이지
내용: 아이디 내용, 로그인 버튼 or 실패시 내용
형태: form태그 
기능: 아이디 내용에서 뒤의 3자리 ***처리, 로그인 버튼->로그인창으로 이동

8.pw찾기 페이지
내용: 아이디, 이름, 이메일, 확인버튼
형태: form태그
기능: 세개의 input입력 후 확인 누르면 pw찾기 결과페이지 이동 / 틀렸을 시 대화상자 출현 후 다시 로그인 페이지로

8-2. pw찾기 결과 페이지
내용 : 메일로 변경된 비밀번호 보냈다는 내용을 or 실패시 내용
기능 : 메일로 난수화된 pw보내기




9.리스트 페이지
내용: 1페이지당 보여지는 수 = 3*3 / 내용 = 상품이미지, 가격, 상품명, 그람수, select형태(최신순, 추천순)
형태: 1페이지당 보여지는 수 = 3*3
기능: 세일된 상품이 있으면 가격변경, 페이징처리(*), javascript 반복문 사용해서 만들어야 함, 각 내용에 상세페이지 링크


10. 상세보기 페이지
내용: 상품이미지, 가격, 상품명, 원산지, 배송관련, 포장타입, 유통기한, 안내사항, 구매수량, 장바구니 버튼
상품이미지, 상품후기(수정, 삭제, 글쓰기), 상품문의(수정, 삭제, 글쓰기)

형태: 상품이미지=좌측상단 / 상품명, 원산지, 배송관련, 포장타입, 유통기한, 안내사항, 구매수량, 장바구니버튼 = 테이블 또는 list형식으로 우측상단
하단 = tab형태로 상품이미지, 상품후기(list), 상품문의(list)

기능: list부분=페이징기능, 아코디언 기능 / tab형태 각각의 tab의 항목을 클릭시 해당 내용의 div로 이동
상품문의 글쓰기 클릭시 작성페이지 이동
상품후기 글쓰기 클릭시 아코디언으로 작성칸 나타내기
상품후기 와 상품문의 제목 클릭시 내용은 아코디언으로 표시 + 자신이 작성한 내용은 수정 및 삭제 버튼 생성
장바구니 클릭시 장바구니 페이지에 제품이 들어가야 한다. + navi에 있는 장바구니 아이콘에 메세지 출력





10-2 상품 후기 작성(상세페이지)
내용 : 제목
형태 : 테이블 아코디언 형식 위쪽(제목-입력/ 작성자-고정 작성날짜-고정) 아래쪽(내용, 사진)
기능 : 사진첨부, 작성완료 버튼 클릭시 내용 저장, 마이페이지에서 작성한 후기 리스트로 출력
여기서 작성한 자신의 후기는 마이페이지의 후기목록에도 나와야 한다.


10-3. 상품문의 작성 & 수정 페이지
내용: 제목, 작성자, 비밀글 선택, 이메일, 핸드폰, 내용, 파일첨부
형태: (작성자, 이메일, 핸드폰, 제목) div / 내용=textArea 또는 글쓰기 api
기능: 작성, 파일첨부, 비밀글 





11.장바구니 페이지
내용: 상품이미지, 상품내용(상품명+단품가격), 수량(+,-) 해당 상품 가격, 전체선택, 각상품의 checkbox, 삭제마크, 주문하기 버튼
형태: 테이블(선택한 상품), div - 전체가격부분
기능: checkbox 변동시 가격 변동






12.주문서 페이지
내용: 주문자 정보(보내는 분, 휴대폰, 이메일), 상품정보(장바구니에서의 내용부분), 배송정보(주소, 배송구분, 수령인 이름, 휴대폰, 배송 요청사항)
결제정보(결제 수단, 동의내용)
형태: div로 각각 나누기 각 파트 안에서는 table 또는 list사용
기능: 상품정보 상세보기 클릭시 내용이 아래로 떨어지면서 출력
배송정보에서 최근배송지(처음일 때는 회원정보에 입력한 정보 / 다음부터는 이전 기록에서 읽어오기) / 새로운 배송지(입력가능 상태 - 주소api) 





13.결제 페이지(api사용 가능)
내용: 주문서 페이지에서 결재수단 선택에 따라 팝업 또는 링크 페이지로 이동 
형태: 
기능: 결제완료 시 => 결제완료 페이지  / 결제실패 시 => 주문서

13-2

14. 주문내역페이지(마이페이지 부분)
내용: 리스트 형태일때 = 해당 결제 내역에서 대표식품 이미지 + 결제정보(금액)+주문상태(임의로)
       상세일때 = 제품 리스트화(제품명, 가격, 후기버튼-배송완료인 경우만 활성), 결제정보, 주문정보, 배송정보

형태: 리스트 형태 -> 상세클릭 시 해당 상품 리스트화 + div-결제정보, div-주문정보, div-배송정보

기능: 후기버튼 특정 조건시(주문상태 = 배송완료)에 활성화, 후기버튼 활성화 때 클릭시 상품후기 작성페이지 이동






15. 상품후기페이지(마이페이지 부분)
내용: 제목, 내용(사진첨부), 수정버튼, 삭제버튼
형태: 제목+사진 = 테이블 형태 / 클릭 시에 아코디언으로 내용 표현 
기능: 클릭시 상품후기 수정페이지로 이동 / 삭제버튼 클릭시 대화메세지창 으로 다시한번 확인 후 최종적으로 내용 삭제


15-2. 상품후기 작성 & 수정페이지
 = 상품문의 작성페이지와 동일한 형태 및 기능(사용되는 변수 및 data만 다르게)
 = 여기서 수정한 내용은 해당 상품 상세보기에 나오는 상품후기 리스트에도 반영이 되어야 한다.





16. 개인정보수정(마이페이지 부분)
내용: 비밀번호 재입력 후 맞으면 회원가입 폼으로 나타난 회원정보 수정 또는 탈퇴페이지 이동 버튼
형태: 회원가입 폼과 동일
기능: 수정버튼 클릭시 수정완료 alert / 탈퇴버튼 클릭시 탈퇴페이지 이동

형태: 
기능: 비밀번호 올바르게 입력 후 탈퇴버튼 클릭 -> alert( )다시 확인

17. 오류페이지(페이지 로딩 또는 무언가 오류가 발생했을 때 404, 또는 500 페이지 대신 나타내주는 페이지)




18. 관리자 로그인시 보이는 페이지(관리자용 마이페이지)
내용 : 회원 관리 / 주문제품 관리 / 상품관리 
회원관리 : 회원리스트(유저명, id, 주소, 핸드폰) 상세(유저명, id, 주소, 생년월일, 성별, 이메일, 구매내역)

상품관리 : 지금 쇼핑몰남은 상품 리스트(+추가버튼 ), 상세(수정가능한 상태) = 관리자가 계속 update를 해줘야 함
